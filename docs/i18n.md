# Internationalization (i18n)

The project uses `next-intl` for complete internationalization support, including routing, translation management, and locale detection.

## Supported Languages

- **English** (`en`) - Default
- **Spanish** (`es`)
- **German** (`de`)

## Structure

All i18n logic is centralized in `src/i18n/`:

```
src/i18n/
├── config.ts          # Locale definitions
├── request.ts         # Request configuration
├── routing.ts         # Navigation helpers
└── locales/           # Translation JSON files
    ├── en.json
    ├── es.json
    └── de.json
```

## Usage in Components

### 1. Translating Text

Use the `useTranslations` hook.

```tsx
import { useTranslations } from "next-intl";

export function Welcome() {
  const t = useTranslations("Home");
  
  return <h1>{t("title")}</h1>;
}
```

### 2. Translation Files

Add keys to `src/i18n/locales/{locale}.json`.

```json
// en.json
{
  "Home": {
    "title": "Welcome to TechWordTranslator"
  }
}
```

### 3. Navigation (Links)

Use the `Link` component from `src/i18n/routing` to automatically handle locale prefixes.

```tsx
import { Link } from "@/i18n/routing";

// Renders /en/about or /es/about depending on current locale
<Link href="/about">About</Link>
```

### 4. Programmatic Navigation

Use `useRouter` from `src/i18n/routing`.

```tsx
import { useRouter } from "@/i18n/routing";

const router = useRouter();

// Navigates to /words preserving locale
router.push("/words");
```

## Adding a New Language

1. **Update Config**: Add the locale code to `src/i18n/routing.ts`.

```typescript
export const routing = defineRouting({
  locales: ['en', 'es', 'de', 'fr'], // Added 'fr'
  defaultLocale: 'en'
});
```

2. **Create Translation File**: Create `src/i18n/locales/fr.json`.

3. **Add Translations**: Copy content from `en.json` and translate values.

4. **Middleware**: Ensure `src/middleware.ts` uses the updated routing config (automatic if importing from `routing.ts`).

## Routing Strategy

- **Prefix-based**: `/{locale}/path` (e.g., `/en/words`, `/es/words`)
- **Default Locale**: `en`
- **Detection**: Automatically detects user preference via `Accept-Language` header.

## Best Practices

- **Namespaces**: Group translations by page or component (e.g., "Home", "Words", "Common").
- **Keys**: Use camelCase for keys.
- **Placeholders**: Use `{variable}` for dynamic content.

```json
"greeting": "Hello, {name}!"
```

```tsx
t("greeting", { name: "User" })
```
